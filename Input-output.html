3. LES ENTREE - SORTIES CONVERSATIONNELLES DU C++

Le C++ hérite toutes les routines de la bibliothèques standard du C, auxquelles s'ajoutent des nouvelles posibilités d'entrées-sorties dont les caractéristiques fondamentales sont:

  • simplicité d'utilisation (absence de notion de format, nécessaire aux fonctions printf et scanf);
  • réduction de la taille du module objet (contrairement à ce qui se passe avec printf et scanf introduisant un ensemble d'instruction couvrant toutes les éventualités, l'emploi des nouvelles possibilités du C++ n'introduisant que les seules intructions nécessaires à leur utilisation);
  • extensibilité de ces possibilités aux types classes.

3-1. Affichage à l'écran

Le C++ est un langage dépourvu des fonctions standard d'entrées-sorties. Cela fait de lui un langage très souple car se servant des bibliothèques pour ses entrée-sorties des fichiers et des résultats.

Exemple 1

printf("Bonjour"); /* seule forme acceptable en C */
cout << "Bonjour"; /* seule forme acceptable en C++ */

Exemple 2

int n=25; cout << "valeur: "; cout << n; /* elles affichent: valeur: 25 */

Les deux instructions précedentes peuvent se condencer en une seule:

cout << "valeur: " << n; /* instruction équivalente à: (cout<<"valeur: ")<< n; */

De manière générale, une intruction telle que:
cout << ------- << ------- << --------< <;
permet d'envoyer sur le flot cout différentes informations, dans l'ordre de leur apparition.

3-2. Possibilités d'écriture sur cout

On peut utiliser l'opérateur << pour envoyer sur le flot cout la valeur d'une expression d'un type de base, de type chaîne ou de pointeur (pour obtenir une adresse). Pour obtenir l'adresse d'une chaine et non les caractères qu'elle contient, on convertit la chaîne (char*) en void*.

3-3. Lecture sur clavier

Comme le flot sortie cout associé à la sortie standard du C (stdout), il existe un flot d'entrés cin, associé à l'entrée standard (stdin). L'opérateur >> permet de recevoir de l'information provenant d'un flot d'entrée.

Exemples:

cin >> n; /* lire des caractères sur le flot de sortie cin et les convertir en int (n étant de type int) */
cin >> n >> p; /* ou (cin cin >> n) >> p; la valeur de n est d'abord extraite du flot cin, pui celle de p du flot cin >> n (le résultat de l'opérateur >> étant un flot. */

3-4. Différentes possibilités de lecture sur cin

Comme avec scanf et getchar, les caractères frappés sont enregistrés dans un tampon, à partir duquel se fera la lecture. Les informations du tampon non exploitées lors d'une lecture restent disponibles pour la lecture suivante. La plupart des conventions d'analyse des caractères lus sont les même que pour scanf:

  • les différentes informations sont séparées par un ou plusieurs séparateurs;
  • un caractère invalide arrête l'exploitation du flot, comme si l'on avait rencontré un séparateur, mais ici, ce caractère sera à nouveau pris en compte lors d'une prochaine lecture.

Contrairement à scanf, la lecture d'un caractère sur cin commence par sauter les séparateurs. Impossible aussi de ire directement ces séparateurs

3-5. Le fichier en-tête iostream

Pour utiliser les fonctions printf ou scanf, le C incorporait le fichier en-tête < stdio.h > contenant des déclarations nécessaires. Quant à C++, les déclarations nécessaires figurent dans le fichier en-tête < iostream.h >
